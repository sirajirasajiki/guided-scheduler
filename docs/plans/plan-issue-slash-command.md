# プラン：/issue コマンドにバグ/タスク判定とテンプレート分岐を追加する

## Context

Issue #5 で GitHub Issue テンプレート（bug / task）の整備が予定されている。
それに合わせて `/issue` スラッシュコマンドも型判定＋テンプレート分岐に対応させる。
バグは再現手順・期待/実際の動作が必要、タスクは背景・やることが必要と構成が異なる。

---

## 変更ファイル

- `~/.claude/commands/issue.md`（グローバル版）
- `.claude/commands/issue.md`（プロジェクト版）

---

## 変更内容

### 型判定ロジック（両ファイル共通）

1. 会話を読んで Issue にすべき内容を特定する
2. バグかタスクかを判定する
   - バグ：「動かない」「エラーが出る」「おかしい」「壊れている」など不具合の報告
   - タスク：「作りたい」「追加したい」「改善したい」「修正したい」など作業の依頼
   - 判断に迷う場合はユーザーに確認する
3. 判定した型に対応するテンプレートで下書きを作成する
4. 下書きをユーザーに提示して承認を得る
5. 承認後に `gh issue create` を実行する

### グローバル版テンプレート

**バグ:** 再現手順 / 期待する動作 / 実際の動作 / 完了条件
`gh issue create --label bug`

**タスク:** 背景 / やること / 完了条件
`gh issue create --label task`

### プロジェクト版テンプレート（グローバル版 + タスク概要 + assignee）

**バグ:** タスク概要 / 再現手順 / 期待する動作 / 実際の動作 / 完了条件
`gh issue create --assignee sirajirasajiki --label bug`

**タスク:** タスク概要 / 背景 / やること / 完了条件
`gh issue create --assignee sirajirasajiki --label task`

---

## ラベルの前提条件

`bug` / `task` ラベルが GitHub 上に存在すること（Issue #5 で作成予定）。
ラベルが未作成の場合は `gh issue create` がエラーになるため、Issue #5 を先に対応すること。
