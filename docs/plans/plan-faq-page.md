# 実装計画: Q&Aページを作成する (Issue #7)

## Context

ユーザーが「データ削除のタイミングが不明確」という疑問を持つケースが発生した。
GitHub Actions (`cleanup-old-events.yml`) で毎月自動削除が走っているが、そのスケジュールをアプリ内で開示できていない。
専用の Q&A ページを作成し、フッターからリンクすることで全ページからアクセス可能にする。

**削除スケジュール（GitHub Actions から）:**
- 実行タイミング: 毎月1日 18:00 UTC（日本時間: 毎月2日 3:00）
- 削除条件: イベントの最新候補日から 30 日以上経過したイベント・参加者データ

---

## 実装方針

専用の `/faq` ページを新設し、フッターにリンクを追加する（全ページから参照可能）。
terms.tsx / privacy.tsx と同じページ構造に倣う。
Q&A項目は配列 + コンポーネントで定義し、追加・編集が容易な設計にする。

---

## 詳細実装

### 1. `src/client/pages/faq.tsx`（新規作成）

- terms.tsx / privacy.tsx と同じラッパー構造を使用
  - `flex-1 bg-gray-50` → `max-w-2xl mx-auto` → 白カード `rounded-2xl shadow`
- Q&A 項目を配列で定義:

```ts
const faqs: { q: string; a: string | JSX.Element }[] = [...]
```

- `faqs.map()` で各 Q&A を描画するシンプルな構造
- Q: 太字・見出し風、A: 通常テキスト（段落区切り対応）

**提案する Q&A 内容（7件）:**

| # | 質問 | 回答の要点 |
|---|---|---|
| 1 | データはいつ削除されますか？ | 毎月1日 18:00 UTC（日本時間 2日 3:00）に、最新の候補日から 30 日以上経過したイベントを自動削除 |
| 2 | 回答を修正・上書きできますか？ | 同じ名前で再回答すると自動的に上書きされる |
| 3 | イベントや参加者データを手動で削除できますか？ | 現在は手動削除機能なし。自動削除を待つか、管理用 URL を第三者に知らせないようにする |
| 4 | 管理用 URL を紛失した場合は？ | 現在は再発行機能なし。作成時のURLを大切に保管してほしい |
| 5 | 確定日を変更できますか？ | 現在は確定後の変更不可。新しいイベントを作り直す必要がある |
| 6 | 何人まで参加できますか？ | 参加人数の上限はなし |
| 7 | このサービスは無料ですか？ | 現在は無料。将来変更する可能性あり（利用規約参照） |

### 2. `src/client/App.tsx`

既存インポート群に `Faq` を追加し、`/faq` ルートを追加:

```tsx
import Faq from './pages/faq';
// ...
<Route path="/faq" element={<Faq />} />
```

### 3. `src/client/components/footer.tsx`

既存の「利用規約」「プライバシーポリシー」リンクに「よくある質問」を追加:

```tsx
<Link to="/faq">よくある質問</Link>
```

フッターは `<a target="_blank">` ではなく `<Link>` を使う（同一 SPA 内遷移）。
既存の terms / privacy リンクも同様に変更検討（現状 `target="_blank"` だが内部ページのため不要）→ ただし既存挙動は変えない。新規リンクのみ `<Link>` で追加。
