### Day 10 トラブル

- 別プロジェクトのディレクトリで Claude Code を起動してしまい、
  既存コードが全く違うものとして認識された
- 教訓: Claude Code 起動前に pwd で現在開いているフォルダの確認する習慣が必要

### 進め方

- 通常起動モード(確認プロンプト都度承認)
- 再度新規でClaude Codeのターミナルを開いた -「CLAUDE.md と既存のコードを読んで、現在の実装状況を理解してください。

理解できたら、機能追加②として以下を実装してください。
2つの機能があり、依存関係があるので順番に実装してください。

【機能1：店への投票機能】

- 参加者が店候補に投票できる(1人1店、複数店可は要検討)
- 管理画面で店ごとの投票数を表示

【機能2：候補日 × 店候補のクロス集計】

- 機能1の投票データと、既存の参加者の日程回答を組み合わせる
- 「この日に来られる人(◯または△)で、この店に投票している人」をマトリクス表示
- 行: 候補日、列: 店候補、セル: 該当する参加者の人数(または名前リスト)

機能1を実装・動作確認してから機能2に着手してください。
両方完了したら報告してください。」
これで機能追加させた。
ただ、店の追加のタイミングが日程追加の後に記載されていたので、追加で指示して機能を変更した。
下記の内容で追加した。
「店の投票タイミングを変更してください。

【現状】
日付の投票 → その後に店の投票（別ステップ）

【変更後】
日付の投票と店の投票を同じ画面で同時にできるようにする。
店候補の横に現在の投票数も表示する。

参加者が1回の送信で、日程回答（◯△×）と店の投票を
まとめて送信できるUIにしてください。」

### Day 10 仕様変更

- Claude Code が作った仕様と人間の意図にズレがあった
- 「投票のタイミング」という暗黙の前提が伝わっていなかった
- 教訓: UIのフロー（何をどの順番で操作するか）は明示的に指示すべき
