会話の流れからバグ・課題・改善案を読み取り、GitHub Issue の下書きを作成してください。

## 手順

1. 直近の会話を読んで、Issue にすべき内容を特定する
2. **バグかタスクかを判定する**
   - バグ：「動かない」「エラーが出る」「おかしい」「壊れている」など不具合の報告
   - タスク：「作りたい」「追加したい」「改善したい」「修正したい」など作業の依頼
   - 判断に迷う場合はユーザーに確認する
3. 判定した型に対応するテンプレートで下書きを作成し、ユーザーに提示する
4. ユーザーが確認・承認したら `gh issue create` で作成する

## Issue フォーマット

### バグの場合

**タイトル:** （端的に不具合を表す一文）

**本文:**
```
## タスク概要

（バグの概要を1〜2文で要約）

## 再現手順

（バグを再現するための手順を箇条書き）

## 期待する動作

（本来こうなるはずという動作）

## 実際の動作

（実際に起きていること）

## 完了条件

（何ができていれば修正完了とみなすか）
```

**コマンド:** `gh issue create --assignee sirajirasajiki --label bug`

### タスクの場合

**タイトル:** （端的に課題を表す一文）

**本文:**
```
## タスク概要

（Issue の目的を1〜2文で要約）

## 背景

（なぜこの課題が生まれたか・どんな問題が起きているか）

## やること

（具体的な対応内容を箇条書き）

## 完了条件

（何ができていれば完了とみなすか）
```

**コマンド:** `gh issue create --assignee sirajirasajiki --label task`

## 注意事項

- 下書きを提示した後、必ずユーザーの承認を待ってから `gh issue create` を実行すること
- `--label bug` / `--label task` は GitHub 上にラベルが存在する前提（未作成の場合はエラーになる）
- リポジトリは現在の git リポジトリから自動判定する（`gh issue create` はリポジトリ指定不要）
- タイトル・本文が会話から読み取れない場合はユーザーに質問する
